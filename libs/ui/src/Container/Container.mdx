# Container

The `Container` component provides configurable sizes.

## Import

```tsx
import { Container } from '@/components/Container'
```

## Usage

### Basic Example

<Canvas of={ContainerStories.Default} />

```tsx
<Container size="md">
  Content
</Container>
```

### Sizes

Available in 5 sizes:

<Canvas of={ContainerStories.Sizes} />

```tsx
<Container size="sm">sm</Container>
<Container size="md">md</Container>
<Container size="lg">lg</Container>
<Container size="xl">xl</Container>
<Container size="full">full</Container>
```

## Props

<ArgTypes of={Container} />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `'sm' \| 'md' \| 'lg' \| 'xl' \| 'full'` | `-` | - |


## Accessibility

The `Container` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Composition

### With Other Components

```tsx
import { Stack } from '@chakra-ui/react'

<Stack spacing={4}>
  <Container>

    Second item
```

## Responsive Design

Use responsive props for different screen sizes:

```tsx
<Container
  size={{ base: 'sm', md: 'md' }}
  width={{ base: '100%', md: 'auto' }}
>
  Responsive Container
</Container>
```

## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
3. **Sizing**: Use consistent sizes within the same section of your UI
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
