# Button

The `Button` component provides multiple visual variants, configurable sizes.

## Usage

### Basic Example

<Canvas of={ButtonStories.Default} />

```tsx
<Button variant="solid" size="md">
  Content
</Button>
```

### Variants

The `Button` supports 3 visual variants:

<Canvas of={ButtonStories.Outline} />
<Canvas of={ButtonStories.Ghost} />

```tsx
<Button variant="solid">solid</Button>
<Button variant="outline">outline</Button>
<Button variant="ghost">ghost</Button>
```

### Sizes

Available in 3 sizes:

<Canvas of={ButtonStories.Sizes} />

```tsx
<Button size="sm">sm</Button>
<Button size="md">md</Button>
<Button size="lg">lg</Button>
```

## Props

<ArgTypes of={ButtonStories} />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'solid' \| 'outline' \| 'ghost'` | `-` | - |
| `size` | `'sm' \| 'md' \| 'lg'` | `-` | - |


## Accessibility

The `Button` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Composition

### With Other Components

```tsx
import { Stack } from '@chakra-ui/react'

<Stack spacing={4}>
  <Button variant="solid">
    First item
  </Button>
  <Button variant="outline">
    Second item
  </Button>
</Stack>
```

## Responsive Design

Use responsive props for different screen sizes:

```tsx
<Button
  size={{ base: 'sm', md: 'md' }}
  width={{ base: '100%', md: 'auto' }}
>
  Responsive Button
</Button>
```

## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
2. **Variants**: Choose the appropriate variant for the context (e.g., primary actions vs secondary)
3. **Sizing**: Use consistent sizes within the same section of your UI
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
