# Badge

The `Badge` component provides multiple visual variants.

## Import

```tsx
import { Badge } from '@/components/Badge'
```

## Usage

### Basic Example

<Canvas of={BadgeStories.Default} />

```tsx
<Badge variant="solid">
  Content
</Badge>
```

### Variants

The `Badge` supports 3 visual variants:

<Canvas of={BadgeStories.Variants} />

```tsx
<Badge variant="solid">solid</Badge>
<Badge variant="outline">outline</Badge>
<Badge variant="subtle">subtle</Badge>
```

## Props

<ArgTypes of={BadgeStories} />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'solid' \| 'outline' \| 'subtle'` | `-` | - |
| `colorScheme` | `'brand' \| 'accent' \| 'success' \| 'slate'` | `-` | - |


## Accessibility

The `Badge` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Composition

### With Other Components

```tsx
import { Stack } from '@chakra-ui/react'

<Stack spacing={4}>
  <Badge variant="solid">
    First item
  </Badge>
  <Badge variant="outline">
    Second item
  </Badge>

```

## Responsive Design

Use responsive props for different screen sizes:

```tsx
<Badge
  
  width={{ base: '100%', md: 'auto' }}
>
  Responsive Badge
</Badge>
```

## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
2. **Variants**: Choose the appropriate variant for the context (e.g., primary actions vs secondary)
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
