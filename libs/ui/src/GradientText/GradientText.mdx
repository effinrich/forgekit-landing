# GradientText

The `GradientText` component provides multiple visual variants.

## Usage

### Basic Example

<Canvas of={GradientTextStories.Primary} />

```tsx
<GradientText variant="primary">
  Content
</GradientText>
```

### Variants

The `GradientText` supports 3 visual variants:

<Canvas of={GradientTextStories.Accent} />
<Canvas of={GradientTextStories.Rainbow} />

```tsx
<GradientText variant="primary">primary</GradientText>
<GradientText variant="accent">accent</GradientText>
<GradientText variant="rainbow">rainbow</GradientText>
```

## Props

<ArgTypes of={GradientTextStories} />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'primary' \| 'accent' \| 'rainbow'` | `-` | - |


## Accessibility

The `GradientText` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Composition

### With Other Components

```tsx
import { Stack } from '@chakra-ui/react'

<Stack spacing={4}>
  <GradientText variant="primary">

  </GradientText>
```

## Responsive Design

Use responsive props for different screen sizes:


## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
2. **Variants**: Choose the appropriate variant for the context (e.g., primary actions vs secondary)
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
